Algorithm Comparison: Min & Max Finder

Brute-Force Presorting Divide & Conquer Comparison Try It

Brute-Force Algorithm

Linear scan of the array to determine the minimum and maximum values.

function bruteForceMinMax(arr) {
  let min = arr[0];
  let max = arr[0];
  for (let i = 1; i < arr.length; i++) {
    if (arr[i] < min) min = arr[i];
    if (arr[i] > max) max = arr[i];
  }
  return {min, max};
}

Presorting-Based Algorithm

Sorts the array and picks the first and last elements as min and max.

function presortMinMax(arr) {
  arr.sort((a, b) => a - b);
  return {min: arr[0], max: arr[arr.length - 1]};
}

Divide-and-Conquer Algorithm

Splits the array into halves, finds min/max recursively and combines the results.

function divideAndConquerMinMax(arr, low = 0, high = arr.length - 1) {
  if (low === high) return {min: arr[low], max: arr[low]};
  if (high === low + 1) {
    return arr[low] < arr[high]
      ? {min: arr[low], max: arr[high]}
      : {min: arr[high], max: arr[low]};
  }
  let mid = Math.floor((low + high) / 2);
  let left = divideAndConquerMinMax(arr, low, mid);
  let right = divideAndConquerMinMax(arr, mid + 1, high);
  return {
    min: Math.min(left.min, right.min),
    max: Math.max(left.max, right.max)
  };
}

Efficiency Comparison

  • Brute-Force: O(n)
  • Presorting-Based: O(n log n)
  • Divide-and-Conquer: O(n)

Use brute-force for small/medium arrays. Divide-and-conquer is optimal in terms of comparisons. Presorting is only useful when sorting is needed for other purposes.
